## v0.7.0 — Pre-release

Third public version of the **Static Code Analysis Tool for Delphi**.
Focus: large frame refactor in the IDE plugin (god-class decoupled),
new **single-file live watch** (replaces WatchMode + AutoScanOnEdit),
overhauled `FormatMismatch` detector, and a standalone HTML export.

### Highlights

- **Single-file live watch** (NEW, replaces WatchMode +
  AutoScanOnEdit) — clicking **Current file** activates a live
  watch on exactly that file: every save (300 ms debounced) AND
  every edit (1000 ms debounced) re-scans the file in the
  background. Switching tabs to another file changes nothing;
  clicking again re-attaches the watch. Bulk paths (**Start
  analysis**, **Branch-Changes**) deactivate the watch explicitly.
  No INI flag any more.
  > ⚠️ **INFINITE-LOOP RISK.** No re-entrancy guard for overlapping
  > worker spawns yet. Before broad use, add a re-entrancy guard +
  > hard cap — see `TODO.md` and the warning block at the top of
  > `uIDEWatchMode.pas`.
- **`FormatMismatch` detector overhauled** — `Format(...)` /
  `FormatUtf8(...)` are now also resolved through **constants** /
  **resource strings** as the first parameter (previously only
  inline literals). Specifier count vs. argument count is matched
  cleanly, with special cases for `%n`, `%%` and indexed
  specifiers.
- **Frame refactor (`TAnalyserFrame` god-class decoupled)** —
  formerly ~2000 lines of frame, now spread across:
  `uIDEAnalyseRunner`, `uIDEAnalyseProgress`,
  `uIDEEditorIntegration`, `uIDEExportMenu`, `uIDEHelpPanel`,
  `uIDEStatusBar`, `uIDEThemeIntegration`, `uIDEStatsTiles`,
  `uIDEGridTooltip`, `uIDELifecycle`. Frame holds only UI
  composition + click delegation, no business logic any more.
- **HTML export** moved into its own unit `uExportHtml` (was
  730+ lines in `uExport` before). New look with Sonar-style tile
  row, severity filter and file-based detail list.
- **`uIDEStatsTiles`** — Sonar-style finding tiles (Total, Critical,
  Warning, Hint per detector group) as a horizontal row above
  the findings grid.

### Removed

- **INI flags `[Detectors] WatchMode` and `AutoScanOnEdit`** — both
  removed without replacement. Live watch is now implicitly tied
  to "Current file" (see Highlights). Existing INI files don't
  need adjustments — the flags are ignored.
- **Legacy `uLeakDetector` / `uStaticAnalyzer` / `uParser`** deleted
  (already replaced by `uLeakDetector2` / `uStaticAnalyzer2` /
  `uParser2` before v0.6.0, now finally removed).

### Configuration

- **`AutoScanOnEdit`** was briefly available in v0.7-rc1 — removed
  again, replaced by single-file live watch.
- **No new INI schema** in v0.7 — all old keys from v0.6 stay valid.

### IDE plugin

- **Frame decomposition** (see above) — every new helper module
  holds its state in its own class instead of as a frame field.
- **Progress-bar marquee mode** during the Branch-Changes diff
  phase (Git/SVN calls have no meaningful progress).
- **Watch module notifier** — `TFindingModuleNotifier` now lists
  all three `IOTAModuleNotifier` / `80` / `90` versions
  explicitly; saving in Delphi 12 otherwise crashes with AV in
  `coreide290.bpl`.
- **Recent-paths MRU** in `uRecentPaths` — last 10 project paths
  in the "Project Path" combo.
- **`uIDELineHighlighter`** robustness fixes — clean
  `RemoveNotifier` even when the editor is already closed.

### Engine

- **`FormatMismatch` const resolution** — map of
  `(UnitName, ConstName) -> value` is built lazily on first
  invocation per method, then consulted for every `Format(...)` /
  `FormatUtf8(...)`. Recognises `Format(MY_CONST_FMT, [...])`
  correctly.
- **`DivByZero`** — better recognition of variable divisors with
  validation earlier in the method (`if x = 0 then Exit` pattern
  reduces false positives).
- **`SQLInjection`** — new safe-cast detection
  (`IntToStr(...) + ' OR 1=1'` doesn't count as injection risk).
- **`uLeakDetector2`** — various FP reductions + new `FieldLeak`
  path for `class.field := TX.Create` without owner.
- **`HardcodedSecret`** — curated filter list for false positives
  (`secretary`, `tokenize`, `passport`, ...).
- **`TodoComment`** — block-comment-aware scanner (no more
  `// TODO` flagged inside `'foo // bar'` string literals).
- **Suppression helpers consolidated** — `noinspection` markers
  are now extracted in a single pass over the token stream
  instead of per detector.
- **`uParser2`** iteration hardening — additional `GuardAdvance`
  spots, `try-finally` block adoption via the new
  `AdoptChildrenFrom` helper (no more O(n²) when re-parenting).

### Tests

- **TestProject** extended to ~270+ cases (`uTestAnalyserChecks`).
- **`uTestSrcBuilder`** helper for compact test source construction.
- **`TestParseX509`** as a reproducer for the mORMot X509 parser
  crash (already fixed; test stays as regression).

### UI / UX

- **DE localisation extensions** in `uLocalization`.
- **Themed findings grid** — VCL-style aware
  (Light / Dark / Mountain Mist).
- **Help panel** (`uIDEHelpPanel`) — inline docs + fix hints in
  the IDE plugin without leaving the frame.
- **Status bar 3-panel** — finding count / file progress / mode
  in the dedicated `uIDEStatusBar` component.

### Robustness

- **Re-entrancy protection** in `uIDEAnalyseRunner` — no double
  start from accidental double-clicks on the analysis button.
- **Path normalisation** in the watch manager — `/` vs `\` and
  case differences are caught in `NormalizePath`, otherwise
  `IOTAModule` paths don't match `EditView.Buffer` paths.
- **Locale-safe `.pas` filter** — `EndsText` instead of
  `ToLower.EndsWith` (Turkish-I problem).
- **Worker generation counter** is also incremented on switch
  between watched files — old worker output no longer lands
  in the new watch context.

### Requirements

- Windows 10/11
- (Plugin) RAD Studio 12 Athens — for the frame refactor +
  ToolsAPI notifier patterns
- (Standalone) none — compiles as a single EXE

### Known limitations

- No binaries in this release — source only. Build the plugin
  (BPL) and standalone (EXE) yourself.
- **Single-file live watch without re-entrancy guard** (see
  Highlights / README). With slow workers + active typing the
  backlog can grow. Workaround: just don't leave it permanently
  on; on performance issues launch a bulk path → watch is
  deactivated.
- Floating-point division (`/`) is not checked by the DivByZero
  detector.
- Plugin unload while a watch worker is running: still possible
  in rare cases an AV occurs (Synchronize into freed memory).
  Workaround: launch a bulk run before unloading the plugin so
  the watch is deactivated.

### Upgrade from v0.6.0

- **`[Detectors] WatchMode` and `AutoScanOnEdit` in `analyser.ini`
  are ignored.** Entries can stay or be deleted — no effect.
  Live watch is now active only via **Current file** click.
- **Frame refactor is UI-transparent** — all buttons, menus,
  shortcuts and finding filters work as in v0.6.
- **Existing suppressions, `ignore.txt` entries, custom
  LeakyClasses and severity configuration** stay valid unchanged.
- **HTML exports from v0.6** are not binary compatible — new look
  + new tile row. No migration path needed, just re-export.
