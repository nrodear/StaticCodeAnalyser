## v0.6.0 ‚Äî Pre-release

Second public version of the **Static Code Analysis Tool for Delphi**.
Focus: deeper IDE integration (editor highlight, watch mode),
end-to-end INI configuration and structural cleanup.

### Highlights

- **Editor line highlight** when clicking a finding ‚Äî 3 px red stripe
  to the left of the line in the IDE editor (via
  `INTAEditViewNotifier`, DelphiLint pattern). Lazy attach on first
  click, clean `RemoveNotifier` on plugin unload.
- **Watch mode** (live analysis on save) ‚Äî `Ctrl+S` triggers a
  re-analysis of exactly the saved file in a background thread,
  300 ms debounce, generation counter drops late worker results.
  Activation via `[Detectors] WatchMode=1`; auto-on for
  "Current file".
- **Auto-discovery of custom classes** ‚Äî `AutoDiscoverClasses=1`
  scans the project AST for classes that need `Free` and splits
  them into _instantiable_ and _static-only_ candidates. Result
  in `LeakyClassesDiscover.log` for manually maintaining the
  `LeakyClasses=` list.
- **Configurable detector thresholds** ‚Äî `LongMethodMaxBodyLines`,
  `LongMethodMaxStatements`, `LongParamListMaxParams`,
  `DeepNestingMaxDepth`, `DuplicateBlockMinLines`, `MaxFileMB`,
  `MagicNumberTrivials` directly in `analyser.ini`, no recompile
  required.

### Configuration

- **`repo.ini` ‚Üí `analyser.ini`** with auto-migration. One central
  INI file for VCS, detector thresholds, custom classes and UI
  language. Created on first start with self-explanatory comments.
- **`UsesCheck` / `IncludeTests` moved from toolbar into the INI** ‚Äî
  toolbar lighter, same pattern as `AutoDiscoverClasses` and
  `WatchMode`.
- **`LeakyClasses` / `ExcludeLeakyClasses`** in `[Detectors]` ‚Äî
  track project-specific classes without code change, or remove
  them from defaults.

### IDE plugin

- **Watch-mode implementation** in `uIDEWatchMode.pas`: per open
  `.pas` an `IOTAModuleNotifier`, all three interface versions
  (`IOTAModuleNotifier` / `80` / `90`) explicitly listed ‚Äî avoids
  an AV in `coreide290.bpl` when Delphi 12 QueryInterfaces the
  90 variant.
- **Line highlighter** in `uIDELineHighlighter.pas`: manager tracks
  `(Notifier, Index, View)` per attach and cleanly calls
  `RemoveNotifier` in the destructor ‚Äî no more AV on plugin
  unload during editor repaint.
- **Toolbar button rename**: "Repo‚Ä¶" ‚Üí "Settings‚Ä¶" /
  "Einstellungen‚Ä¶", now opens `analyser.ini` directly.
- **Tooltip in the grid** (file column only, 100 ms delay instead
  of the IDE default 500 ms).
- **Severity / type filter combos** packed into separate container
  panels ‚Äî fixes the mixed behaviour between graphic and window
  control on `alLeft` layout.

### Engine

- **Restructuring** of the `sources/` hierarchy into
  `Common/`, `Parsing/`, `Detectors/`, `Infrastructure/`,
  `Output/`, `UI/`.
- **`TDetectorUtils` word-boundary helpers** ‚Äî shared by five
  detectors, replaces several ad-hoc implementations.
- **Iterative AST traversal** throughout ‚Äî no more stack overflow
  on deeply nested code.
- **JSON / HTML encoding RFC-compliant** with surrogate handling.
- **Suppression now covers all 21 finding kinds** (gaps before).
- **UTF-8 BOM export** via the `TExporter.SaveUtf8WithBom` helper ‚Äî
  required for German Excel; the default singleton in Delphi 12
  has `FUseBOM=False`.
- **FixHint texts now fully in English** and extended with
  alternative solutions (description / before / after) ‚Äî fits
  code reviews, Jira tickets and Claude AI prompts in practice.
- **Consolidated `uClaudePrompt` helper** ‚Äî no more 1:1 duplicate
  code between standalone and plugin.
- **DE localisation of the UI** (built-in dictionary in
  `uLocalization`); default is English, switchable via INI.

### Robustness

- **UI race protection**: global `GLiveAnalyserFrame` sentinel
  prevents AV on frame destruction during a worker callback.
- **`FilterCombo` edge cases**: `Items.Count = 0` + `idx >= Count`
  pre-checks plus re-entry protection on ItemIndex reset.
- **Watchdog: 200k tokens per file** ‚Äî pathological input is
  aborted in under a second.
- **`GuardAdvance`** in every outer parser loop ‚Äî no more
  infinite loops on malformed input.

### Requirements

- Windows 10/11
- (Plugin) RAD Studio 12 Athens
- (Standalone) none ‚Äî compiles as a single EXE

### Known limitations

- No binaries in this release ‚Äî source only. Build the plugin
  (BPL) and standalone (EXE) yourself.
- Floating-point division (`/`) is not checked by the DivByZero
  detector.
- Watch mode attaches notifiers only on activation ‚Äî modules
  opened _afterwards_ don't trigger live analysis until a new
  run starts.
- Plugin unload while a watch worker is running can in rare cases
  trigger an AV (Synchronize into freed memory). Workaround: turn
  off WatchMode in the INI before removing the plugin.

### Upgrade from v0.5.0

- `repo.ini` is automatically migrated to `analyser.ini` on first
  start.
- Toolbar checkboxes `with uses check` / `Include tests` are gone ‚Äî
  values now live under `[Detectors] UsesCheck=` and
  `IncludeTests=`.
- Existing suppressions, ignore.txt entries and severity
  configuration remain unchanged.
