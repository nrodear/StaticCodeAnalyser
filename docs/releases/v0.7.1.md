## v0.7.1 — Pre-release

Patch release on top of v0.7.0 focused on code metrics (new
detector), responsive docked-mode UI for the IDE plugin,
AI-prompt improvements and end-to-end DPI scaling.

### Highlights

- **Cyclomatic Complexity (McCabe) detector** (NEW, phase 1 of the
  code-metrics roadmap) — `TCyclomaticComplexityDetector` measures
  per method the McCabe complexity: 1 base + `if` + `case` arm +
  `for` / `while` / `repeat` + `on` handler + `and` / `or` / `xor`
  binary ops. `else` does not count (binary branch). Default
  threshold: 10 (Sonar/Checkstyle/PMD standard) via
  `[Detectors] CyclomaticMax`. Findings as `fkCyclomaticComplexity`
  with refactor hint in `uFixHint` (before/after example with
  `CanProcess` / `ProcessOrder` method split).
- **Docked-mode UI** (NEW, IDE plugin) — the plugin frame detects
  narrow containers (< 700 px ClientWidth, typical for docked
  IDE tool panels) and reduces itself to the essentials:
  - Stats tiles shrink to 4 essential ones (Errors / Warnings /
    Hints / Code Quality)
  - Action buttons (Start / Current / Branch-Changes) move into the
    **hamburger menu** (☰)
  - Settings / ignore list also reachable via hamburger
  - Filter combo labels disappear (combos are self-explanatory)
  - SearchEdit min-width shrinks from 120 to 60 px in docked mode
  - Sub-panel containers (Severity / Type combo) adjust their width
    to match label visibility
  - **Floated**: everything as before — full feature set visible
- **AI prompt rewrite** (`uClaudePrompt`) — role priming up front
  ("senior Delphi developer reviewing static-analysis output"),
  structured response template (Cause / Fix / Verify), before/after
  marked as "Reference pattern (NOT the user's code)", source
  marker `>>> ` made more visible, false-positive path with a
  `// noinspection` suggestion. Fully localised through `_(...)` —
  DE users get a DE prompt → DE AI response.
- **DPI scaling** end-to-end in the IDE plugin UI — all hardcoded
  pixel widths/heights now go through `ScaleW(...)` (frame) or
  `ScaleByPPI(...)` (stats tiles). Plugin no longer looks half-size
  on 200% DPI. The docked threshold of 700 is also scaled.

### IDE plugin

- **Cyclomatic stats tile** between "Duplicates" and "Code Quality"
  (magenta branch glyph, ICON_SMELL hue). Counts
  `fkCyclomaticComplexity` findings.
- **Clickable stats tiles** — hover shows a multi-line tooltip with
  an explanation; click filters the grid to the corresponding
  severity or type bucket. Code-quality tile resets all filters.
- **Filter combo** gets a "Cyclomatic Complexity" entry.
- **`TResponsiveVisibilityController`** (in `uIDEStatsTiles`) —
  generic layout controller, hooks Parent.OnResize chained without
  destroying existing handlers, AInverse flag for
  "show-only-when-narrow" controls (hamburger).
- **`TAnalyserFrame.Resize` override** (instead of just OnResize
  event) — guarantees responsive trigger even on Float→Dock
  transitions, where the IDE dock logic sometimes swallows
  the OnResize event.
- **Magic numbers consolidated** into `BTN_W_*` / `LBL_W_*` /
  `CMB_W_*` / `STATS_*` / `TB_*` constants. 14 scattered pixel
  values previously, now a named const section.
- **`BuildHamburgerMenu`** extracted as its own method (was ~25
  inline lines).

### Engine

- **Cyclomatic detector implementation** reads `nkIfStmt.TypeRef`
  for boolean-op counts (the parser doesn't build an expression
  AST, only the cond text on if nodes). `while` / `repeat` / `case`
  conditions aren't available at all — acceptable trade-off vs.
  parser extension.

### Tests

- **`uTestAnalyserChecks` (7500 LOC) split** into 16 per-detector
  test units: `uTestFindingHelper` (shared `TFindingHelper`), plus
  one unit per detector theme group (`uTestLeakDetector`,
  `uTestSQLInjection`, `uTestHardcodedSecret`, ...). DUnitX
  `initialization` block deleted; runner uses RTTI
  (`UseRTTI := True`) and finds `[TestFixture]` classes
  automatically once the unit is referenced.
- **`TTestCyclomaticComplexity`** with 10 new tests in
  `uTestCodeMetrics` (Trivial, SingleIf, ElseDoesNotCount,
  BooleanAndOr, ManyIfs, CaseArms, ForWhileRepeat, OnHandler,
  TryFinallyNotCounted, TwoMethodsOneOver).

### i18n

- **~40 new dictionary entries** in `uLocalization`: tile hints
  (multi-line, EN source → DE dict), hamburger menu items,
  cyclomatic tile caption + filter combo entry, AI prompt headings
  + structured-response building blocks, watch status, cyclomatic
  fix hint.
- **`uIDEWatchMode`** status strings replaced with `Format(_(...), [...])`
  — were hardcoded English without `_(...)` wrap before.
- **Stale dict entries** fixed: `Settings: ... next click of
  Branch-Changes` → `... the next analysis run` (source had been
  changed, dict still showed the old variant). Spelling drift fix:
  `Analyzing:` (US) → `Analysing:` (BR), now matches the existing
  dict key.

### Configuration

- **`[Detectors] CyclomaticMax=10`** (NEW, default 10) — INI doc
  block extended with explanation of what counts.

### Known limitations

- **Single-file live watch without re-entrancy guard** — unchanged
  from v0.7.0. With slow workers + active typing the backlog can
  grow. Phase 2 (TODO.md _Single-File-Live-Watch_).
- **Docked-mode UI** is phase 1 of the roadmap — further phases
  in `TODO.md` (layout architecture via TFlowPanel, two-mode UI,
  polish).

### Upgrade from v0.7.0

- **`[Detectors] CyclomaticMax=10`** is new in the default INI
  template, but existing `analyser.ini` files without that entry
  simply pick up the default. No migration path needed.
- **IDE plugin layout** changes on resize/dock — floated mode
  visually like v0.7.0, docked mode is new (hamburger + reduced
  toolbar). No user action required.
- **Existing suppressions, `ignore.txt` entries, custom LeakyClasses,
  severity configuration** stay valid unchanged.
- **HTML exports** unchanged.
