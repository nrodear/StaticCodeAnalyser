# Release 0.9.8 — Phase 1-4 + Hardening v3/v4 + FP-Reduktion-Sprint

**Released:** 2026-06-10
🇬🇧 [English version](v0.9.8.md)

Dieses Release schliesst den 0.9.8-Zyklus ab, der am 2026-06-02 begann.
Drei Arbeitsschichten sind seit v0.9.7 hinzugekommen:

1. **Scanner-Qualitaet Phasen 1-4** (seit v0.9.7) — 13 Quick-Wins,
   Confidence-Audit, A.5 `{$IFDEF}`-Awareness, SCA165 UnusedSuppression,
   Golden-Corpus-Regression-Tests, SARIF `partialFingerprints` fuer
   Baseline-Diff.
2. **Hardening v3 + v4** (2026-06-07/08) — Stack-Overflow-Schutz fuer
   tiefe ASTs (8 iterative-DFS Detector-Refactors + PE-Patch auf 32 MB
   Stack-Reserve), AST-Destroy-Reentrancy-Double-Free-Fix, DFM-Resource-
   Wrapper (`$FF $0A $00`)-Support, IDE-Plugin Win32-OOM-Fix via
   FixHint-Memoize-Cache, scan.log Phase-Tracking + Skip-Log.
3. **FP-Reduktion-Sprint** (2026-06-09) — Self-Scan-FPs in vier Style-
   Detektoren um 78% reduziert (67 -> 15). Nebenbei-Fix:
   `FreeAndNil(Self.Field)` mit `Self.`-Qualifier wird jetzt als
   Freigabe erkannt.

Real-World-Verifikation: 24 Repos in `D:\git-sca-realworld\` scannen
in 411 s mit **0 Crashes** und produzieren 1.055.283 Findings bei
Profile `strict`.

## TL;DR Highlights (zusaetzlich zum urspruenglichen Phase-1-Plan)

- **DFM Resource-Wrapper-Format `$FF $0A $00`** wird jetzt korrekt
  geparst — GExperts springt von 0 auf 1.084 DFM-Findings, JVCL-DFM-
  Coverage verdoppelt sich. Vorher wurde das Format faelschlich als
  Text interpretiert und still ignoriert.
- **Kein `EInvalidPointer`/SCA006-Crash mehr** mitten im Scan via
  AST-Destroy-Reentrancy-Fix (`5ecd498`).
- **Kein Win32-`EOutOfMemory` mehr** im IDE-Plugin-`HighlightAllFindingsInFile`
  bei ≥100k Findings — `TFixHintResolver` cached jetzt nach
  `(Kind, Severity)`.
- **scan.log Phase-Tracking** — jeder `Analyseabbruch:`-Eintrag zeigt
  jetzt die letzte erfolgreiche Phase + das aktuelle File im Log,
  statt der alten Black-Box.
- **scan.log Skip-Log** — ignorierte / ausgeschlossene Files erscheinen
  mit Grund, statt stillschweigend zu verschwinden.
- **4 Style-Detector-FP-Klassen** behoben ohne echte-Bug-Erkennung zu
  schwaechen: `SCA017 DebugOutput` (String-Literal-Inhalte), `SCA070
  CommentedOutCode` (Inline-Doku + Doc-Block-Start), `SCA019 TodoComment`
  (Detector-Source-Erwaehnungen), `SCA005 FormatMismatch` (leerer Const-
  Resolve).
- **FieldLeak-Detector** — `FreeAndNil(Self.Field)` mit
  `Self.`-Qualifier wird jetzt als Freigabe erkannt (USepa.pas-Repro).
- **Zwei neue Konfig-Optionen** —
  `[Detectors] MaxLineLength=N` (Default 120) und
  `[Detectors] MaxCaseBranches=N` (Default 10).
- **Anfaenger-Build-Guide** — `HowTo_Build{,_de}.md` fuehrt
  Delphi-Neulinge von der IDE-Installation bis zum ersten
  IDE-Plugin-Scan.

## Phase 1 Quick-Wins (6/6)

### C.4 Per-Detektor-Performance-Profil

`--time-detectors` schreibt eine Markdown-Tabelle mit kumulierter
Wall-Time, Call-Count und Mittelwert pro Detektor. Datenbasierte
Auswahl wo es sich zu optimieren lohnt.

```pwsh
analyser.exe --path D:\repo --full --time-detectors > perf.md
```

### A.2 Test-Fixture-Auto-Detection

`TDetectorUtils.IsTestFixturePath` liefert `true` fuer Pfade die matchen:

- Basenames: `uTest*.pas`, `*_Test.pas`, `*_Tests.pas`, `*Sample.pas`,
  `*Demo.pas`, `*Sample_*.pas`, `*_Demo_*.pas`, `*Sample.dfm`,
  `*Demo.dfm`, `MeineUnit.pas` (+ einige SCA-interne Demo-Files)
- Repo-Root-relative Pfad-Komponenten:
  `test`, `tests`, `unittest`, `unittests`, `samples`, `demos`,
  `resources`

Auto-On fuer `--profile default` und `selftest-quiet`, Off fuer `strict`.
Manueller Override via `--hide-test-fixtures` / `--show-test-fixtures`.

### C.3 Unused-Suppression-Tracking (SCA165)

Fuer jeden `// noinspection X`-Marker tracken wir initial
`Consumed = false`. Wenn `ApplyToFindings` ein Finding via diesen Marker
unterdrueckt, `Consumed := true`. Nach der Suppression-Phase emittiert
jeder Marker mit `Consumed = false` ein `fkUnusedSuppression`-Finding
auf der Marker-Zeile mit Quick-Fix-Hinweis zum Entfernen.

### C.1 Golden-Corpus-Regression-Tests

`tests/golden-corpus/fp-reproducers/` enthaelt 5 historische False-
Positive-Reproducer-`.pas`-Snippets (einen pro Round-1..13-Fix).
`expected.json` mappt jedes File auf eine `must_not_flag`-Liste von
Rule-IDs. Der PowerShell-Runner `tools/check-golden-corpus.ps1` scant
den Corpus und exit 0/1 je nachdem ob eine frueher gefixte FP-Klasse
wieder triggert.

```pwsh
powershell -ExecutionPolicy Bypass -File tools\check-golden-corpus.ps1
```

### C.2 SARIF `partialFingerprints.contextHash/v1`

Neue Unit `uFindingFingerprint`:

```
contextHash/v1  =  "v1:" + SHA256( Normalize( +/-3 Zeilen um die Fund-Stelle ) )
```

`Normalize` kollabiert Whitespace-Runs zu einem Space, Tabs → Space,
leere Zeilen entfernt, CRLF / LF normalisiert. Der Hash ist stabil
gegen Re-Indent, Trailing-Whitespace-Cleanup und kleine Line-Drift-
Refactors.

`uBaseline.Apply` matched `contextHash` **oder** Legacy-Fingerprint —
alte Baselines ohne `contextHash` bleiben gueltig (Backward-Compat),
neue Baselines ueberleben kleine Refactors.

### A.1 Confidence-Tagging-Audit

`KindDefaultConfidence` in `uSCAConsts` taggt ~35 Detektor-Kinds als
`fcMedium` (Pattern-Match, Metriken, Style, DFM-Heuristik, Security
ohne Datenfluss). Alle strukturellen Bug-Detektoren bleiben `fcHigh`.
`TLeakFinding.SetKind` zieht den Default-Wert aus dieser Map. Override-
Pattern (z.B. `uCommandInjection := fcLow`) bleibt erhalten via neuen
`SetKind(K, AConfidence)`-Overload. Per-Kind-Begruendungen in
[docs/ConfidenceAudit.md](../ConfidenceAudit.md).

## Phase 4 partial — A.3 Cross-Unit-Visibility (begonnen)

### Minimaler Schritt

`uVisibilityCheck` fuer `fkUnusedPublicMember` (SCA052) konsultiert
jetzt `gSymbolRefIndex.HasExternalRefs` bevor er das Finding emittiert.
Wenn irgendeine externe Unit den Member per `Obj.Member(args)`
referenziert, wird kein Finding mehr emittiert.

### Audit (Spot-Check 18 bekannte Cross-Unit-Methoden)

- ✅ **8/18 (44 %) A.3 greift** — Pattern: `instance.Method(args)`
- ❌ **10/18 (56 %) A.3 wirkt nicht** — 3 bekannte Index-Limitationen:
  - **nkRef / Value-Use ohne Klammern** (`F.SeverityText`)
  - **Class-Function-Call** `TClass.ClassMethod(args)` — AST klassifiziert
    es nicht als `Obj.Member`-Pattern
  - **Bare-Calls** (`Fingerprint(F)`) — bewusste Index-Vereinfachung

Alle drei sind in `Konzept_ScannerQualitaet.md §A.3+` als priorisierter
Follow-Up-Sprint dokumentiert.

### Library-FP-Klasse (nicht per Index loesbar)

Real-World-Scan zeigt: Top-SCA052-Hotspots sind Vendor-Libraries
(`mormot.*`, `MVCFramework`, `LoggerPro`) deren Konsumenten in anderen
Repos leben, die der Index nicht sehen kann. Empfehlung: Path-Override-
Default fuer `vendor/`-Ordner.

## Multi-Persona-Review

Nach Phase 1 + A.3-Minimal wurde ein Drei-Personae-Review (Architektur
+ Security + Performance) auf den Branch losgelassen. 9 von 10 Findings
wurden in Commit 120894a umgesetzt:

| # | Bereich | Fix |
|---|---------|-----|
| 1 | Perf | `Baseline.Apply` skip `ContextHash` bei Legacy-Baseline |
| 2 | Sec | `noinspection All` schliesst Security-Critical-Kinds aus |
| 3 | Sec | `ParseMarkerLine` nutzt `ScanCodeLine` (String-Context-aware) |
| 4 | Sec | `IsTestFixturePath` Repo-Root-anchored via `BaseDir` |
| 5 | Perf | `gFileTextCache` lebt durch Post-Scan + Leak-Fix |
| 6 | Sec | Baseline-JSON 1 M-Entry / 256-Char-Cap |
| 8 | Perf | Suppression nutzt `AcquireLines` (Cache-Hits) |
| 9 | Perf | `uVisibilityCheck`-Tree-Walks gecacht pro Unit |
| 10 | API | `SetKind(K, AConfidence)`-Overload |

Finding #7 (DefaultConfidence in `TFindingKindMeta`-Record) verschoben
weil invasiv (~160 `KIND_META`-Eintraege beruehren) und ein expliziter
Drift-Test-Design noetig waere.

`TFileTextCache` wurde anschliessend mtime-aware gemacht (Commit
1e7e193) um ein Stale-Cache-Issue zu fixen, das die Regression-Suite
nach der Post-Scan-Cache-Aenderung aufgedeckt hatte.

## Migration

Keine Breaking-Changes fuer Caller der Public-CLI / SARIF / Baseline.

- Bestehende Baselines funktionieren wie bisher (matched via Legacy-
  Fingerprint).
- Neue Baselines tragen zusaetzlich `contextHash`. Alte `Baseline.Apply`-
  Versionen ignorieren das unbekannte Feld.
- Detektoren die vorher `F.Confidence := xxx` NACH `SetKind` gesetzt
  haben, sollten auf `SetKind(K, AConfidence)` migrieren um das
  Reihenfolge-Anti-Pattern zu vermeiden. Das alte Pattern bleibt
  kompatibel.
