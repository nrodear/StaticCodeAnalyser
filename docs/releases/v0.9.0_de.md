## v0.9.0 — Silent-Modus, Tools>Optionen-Seite, Regelsatz-Profile, IDE-Plugin-Politur

Workflow-fokussiertes Release mit drei großen Themen:

1. **Silent-Modus** — Einzeldatei-Analyse direkt aus dem Editor-
   Rechtsklick + `Strg+Alt+A`-Hotkey. Kein Dock-Fenster wird
   geöffnet. Befunde erscheinen als Gutter-Marker + Hover-Overlays
   im Code-Editor.
2. **Regelsatz-Profile** — `ide-fast`, `default`, `strict`,
   `security`, `bugs-only`, `code-quality`, `dfm-only` als
   JSON-Katalog (`rules/sca-rules.json`) plus CLI-Flags `--profile`
   / `--min-severity` für CI-Integration.
3. **Tools > Optionen-Seite** unter *Third Party > Static Code
   Analyser* in der Delphi-IDE — Silent-Modus-Schalter,
   Profil-Dropdowns, Detektor-Toggles. Alles in `analyser.ini`
   persistiert.

## 🤫 Silent-Modus (NEU)

Ein neuer Analyse-Pfad der das Dock-Fenster komplett umgeht:

- **Editor-Rechtsklick → "Aktuelle Datei analysieren (Silent)"** —
  analysiert die gerade aktive `.pas` und zeigt Befunde inline.
- **`Strg+Alt+A`-Hotkey** — gleiche Aktion, editor-weit.
  Registriert als echte `IOTAKeyboardBinding`, funktioniert ohne
  vorheriges Popup-Öffnen.
- **Inline-Darstellung** — Gutter-Marker + `[✕]`-schließbare
  Hover-Overlays pro Befund. Marker überleben Tab-Wechsel dank
  Multi-File-Marker-Storage, werden beim Speichern automatisch
  gelöscht (Zeilennummern stimmen nach Edits nicht mehr).
- **Abschalt-Toggle** — `Tools > Optionen > Third Party > Static
  Code Analyser > Silent-Modus`. INI: `[Silent] Enabled=0/1`.

Implementierungs-Notizen:

- Editor-Popup-Integration nutzt das **OnPopup-Chain-Pattern**
  (GExperts/CnPack-Stil): bei jedem Popup-Show wird unser
  vorheriges Item entfernt, dann lassen wir die IDE das Menü
  rebuilden und hängen ein frisches Item ans Ende. Permanente
  Inserts kollidieren mit dem Action-Manager-Rebuild der IDE.
- `IOTAKeyboardBinding` registriert `Strg+Alt+A` editor-weit, so
  funktioniert der Hotkey auch ohne vorherigen Rechtsklick.
  Diagnose-Logging bei Registrierungs-Konflikten (andere Plugins
  die denselben Shortcut nutzen).

## 🎚 Regelsatz-Profile (NEU)

Sieben mitgelieferte Profile decken die gängigen Analyse-Szenarien
ab:

| Profil | Use-Case |
|--------|----------|
| `ide-fast` | Default für IDE-Plugin Silent-Modus — schnelles Subset für Live-Feedback |
| `default` | Standalone-GUI-Default — vollständiges Detektor-Set |
| `strict` | CI-Gate — alles an, inklusive stilistischer Detektoren |
| `security` | OWASP / Vulnerability-Fokus |
| `bugs-only` | Nur `Bug`-Typ-Befunde (kein Smell, kein Style) |
| `code-quality` | Smells + Cyclomatic + Duplikate |
| `dfm-only` | Nur die 20 DFM-Detektoren |

Quelle: `rules/sca-rules.json` — Single Source of Truth. Delphi-
IDE-Plugin und CLI lesen denselben Katalog.

CLI-Flags für CI:

```bash
analyser.d12.exe path/to/repo --profile strict --min-severity warning --format sarif
```

INI-Keys für persistente Konfiguration:

```ini
[Rules]
Profile=strict          # Standalone + CLI
IdeProfile=ide-fast     # IDE-Plugin (Dock-Fenster + Silent-Modus)
MinSeverity=warning
```

## ⚙ Tools > Optionen-Seite (NEU)

Das Delphi-IDE-Plugin bringt jetzt eine Konfigurationsseite unter
**Tools > Optionen > Third Party > Static Code Analyser** mit:

- **Silent-Modus** — Editor-Rechtsklick + Hotkey ein/aus
- **Regelsatz** — Profil (CLI/Form), Min-Schweregrad, IDE-Profil
- **Detektoren** — UsesCheck, IncludeTests, AutoDiscoverClasses

Alle Werte werden in dieselbe `analyser.ini` geschrieben, die auch
das Dock-Fenster nutzt — die Optionen-Seite ist also nur eine
alternative UI, kein doppelter Settings-Store. Seitenöffnen ist
schnell (~5 ms): `TMemIniFile` + vorgewärmter `TRuleCatalog`.

Die Profile-Combo im Dock-Fenster wirkt zusätzlich als Live-
Override und gewinnt über die INI für Silent-Modus-Runs — du
kannst im Dock zwischen Profilen wechseln und der nächste Silent-
Modus-Trigger nimmt das ohne Save-Druck auf.

## ✨ IDE-Plugin-Politur

- **Annotation-Overlay** — `[✕]`-Button pro Hover-Overlay zum
  Entfernen einzelner veralteter Befunde (zusätzlich zum
  datei-weiten Reset beim Speichern). After-Code-Sektion im
  Help-Panel kommt mit `✓`-Marker-Prefix.
- **Multi-File-Marker-Storage** — `TFindingHighlighter` keyed
  Marker per `NormalizePath(filename)`, so dass Tab-Wechsel
  Markierungen anderer Dateien nicht mehr löscht.
- **Auto-Clear beim Speichern** — `IOTAModuleNotifier` pro
  markierter Datei angehängt; `AfterSave` löscht die Marker
  dieser Datei (Zeilennummern stimmen nicht mehr nach Edits).
- **Performance** — `TIniFile` → `TMemIniFile` (~25 Syscalls → 1)
  in `uRepoSettings`. `TRuleCatalog.ProfileNames` beim Plugin-Load
  vorgewärmt, damit das erste Tools>Optionen-Render nicht die
  ~20-50 ms JSON-Parse-Kosten zahlt.

## 🖥 Standalone-Form

UI-Parität mit dem IDE-Plugin: Toolbar mit Toolbar-Buttons, Stat-
Tiles-Reihe oberhalb des Grids, Hint-Panel rechts vom Grid für
Vorher/Nachher-Code-Beispiele, Drei-Panel-StatusBar. Profile- +
Min-Schweregrad-Combos in der Toolbar werden aus
`TRuleCatalog.ProfileNames` befüllt. Die Standalone-EXE öffnet
beim Start kein schwarzes Konsolen-Fenster mehr (war
`{$APPTYPE CONSOLE}` aus dem alten DPR-Template; jetzt
GUI-Subsystem mit `AttachConsole` für den optionalen CLI-Pfad).

## 🐛 Fixes

- **Silent-Modus `AutoDiscoverCustomClasses`** — das globale Flag
  wird jetzt vor jedem Silent-Run aus `Settings.AutoDiscoverClasses`
  gespiegelt. Vorher lief die Detection mit dem State den der
  letzte Dock-Run hinterließ, oder `False` beim Kalt-Start.
  `DiscoveredClasses` / `DiscoveredStaticClasses` werden pro
  Silent-Run gelöscht, damit Treffer aus einem anderen Projekt
  nicht in die aktuelle Analyse bluten.
- **`FEditorEventsObj` MemoryLeak False-Positive** — `// noinspection
  MemoryLeak`-Marker direkt über die `.Create`-Zeile in
  `uIDELineHighlighter` gesetzt. Der Detektor flaggt das als Leak,
  aber `TFindingEditorEvents` erbt `TInterfacedObject` und der
  Refcount wird über `FEditorEvents : INTACodeEditorEvents`
  gehalten. Kein expliziter `Free` nötig.
- **`uDfmRepoIndex` / `uFormBinder` H2077-Hinweise** — Dead-Store-
  `nil`-Zuweisungen vor Re-Assignment entfernt.
