# SCA034 — Required field has no UI binding

![Severity](https://img.shields.io/badge/severity-Warning-orange) ![Type](https://img.shields.io/badge/type-Bug-red)

> TField with Required=True has no bound input control

## Metadata

| Field | Value |
|---|---|
| **Rule ID** | `SCA034` |
| **Kind** | `DfmRequiredFieldUnbound` |
| **Default severity** | Warning |
| **Type** | Bug |
| **Detector unit** | `uDfmRequiredField.pas` |
| **Tags** | `dfm`, `ux`, `required-field` |

## Description

A required field that the user cannot reach makes every insert fail with 'Field X must have a value'. Either bind a control or drop the Required=True.

## Examples

### Bad (triggers the rule)

```pascal
FieldDefs: Email Required=True   // no DBEdit / DBLookup bound to it
```

### Good (idiomatic fix)

```pascal
(bind a DBEdit to the Email field)
```
