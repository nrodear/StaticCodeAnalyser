# SCA024 — Component with default name

![Severity](https://img.shields.io/badge/severity-Hint-blue) ![Type](https://img.shields.io/badge/type-Code%20Smell-yellow)

> Component left at wizard-default name (Button1, Edit3, Panel2 ...)

## Metadata

| Field | Value |
|---|---|
| **Rule ID** | `SCA024` |
| **Kind** | `DfmDefaultName` |
| **Default severity** | Hint |
| **Type** | Code Smell |
| **Detector unit** | `uDfmDefaultName.pas` |
| **Tags** | `dfm`, `naming` |

## Description

Default names hide intent and break find-usages / rename refactorings. Rename UI controls to convey purpose (btnSave, edUserName, pnlToolbar ...).

## Examples

### Bad (triggers the rule)

```pascal
object Button1: TButton
  Caption = 'Save'
end
```

### Good (idiomatic fix)

```pascal
object btnSave: TButton
  Caption = 'Save'
end
```
