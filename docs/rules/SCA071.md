# SCA071 — Unit-level keyword not at column 1

![Severity](https://img.shields.io/badge/severity-Hint-blue) ![Type](https://img.shields.io/badge/type-Code%20Smell-yellow)

> `unit`/`interface`/`implementation`/`initialization`/`finalization` should start at column 1

## Metadata

| Field | Value |
|---|---|
| **Rule ID** | `SCA071` |
| **Kind** | `UnitLevelKeywordIndent` |

| **Type** | Code Smell |
| **Detector unit** | `uUnitLevelKeywordIndent.pas` |
| **Tags** | `style`, `structure` |

## Description

Object Pascal convention is that the structural section keywords of a unit are flush-left at column 1. Indented section keywords obscure the unit's structure when scanning and may hint at miscounted begin/end nesting. Matches SonarDelphi communitydelphi:UnitLevelKeywordIndentation.

## Examples

### Bad (triggers the rule)

```pascal
  implementation

  procedure Foo; ...
```

### Good (idiomatic fix)

```pascal
implementation

procedure Foo; ...
```
