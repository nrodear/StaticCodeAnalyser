# SCA025 — Hardcoded UI text in DFM

![Severity](https://img.shields.io/badge/severity-Hint-blue) ![Type](https://img.shields.io/badge/type-Code%20Smell-yellow)

> Caption / Hint / Text property as literal in DFM, not via i18n layer

## Metadata

| Field | Value |
|---|---|
| **Rule ID** | `SCA025` |
| **Kind** | `DfmHardcodedCaption` |
| **Default severity** | Hint |
| **Type** | Code Smell |
| **Detector unit** | `uDfmHardcodedCaption.pas` |
| **Tags** | `dfm`, `i18n`, `localization` |

## Description

User-facing strings embedded in a .dfm cannot be localised, A/B-tested, or kept in a translation catalog. Assign at form construction time from a resourcestring or i18n helper.

## Examples

### Bad (triggers the rule)

```pascal
object btnSave: TButton
  Caption = 'Speichern'
end
```

### Good (idiomatic fix)

```pascal
btnSave.Caption := _('btn.save');  // i18n at runtime
```
