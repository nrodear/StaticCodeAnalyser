# SCA041 — Input control directly on TForm

![Severity](https://img.shields.io/badge/severity-Hint-blue) ![Type](https://img.shields.io/badge/type-Code%20Smell-yellow)

> Input control sits on the form instead of being embedded in a TPanel / TGroupBox

## Metadata

| Field | Value |
|---|---|
| **Rule ID** | `SCA041` |
| **Kind** | `DfmLayerViolation` |
| **Default severity** | Hint |
| **Type** | Code Smell |
| **Detector unit** | `uDfmLayerViolation.pas` |
| **Tags** | `dfm`, `layout` |

## Description

Layered layout (Form > Panel > Group > Controls) makes resizing, DPI-scaling, and theming significantly easier. Wrap controls in a layout container.

## Examples

### Bad (triggers the rule)

```pascal
object frmEdit: TForm
  object EdName: TEdit  // direct child of form
```

### Good (idiomatic fix)

```pascal
object frmEdit: TForm
  object pnlBody: TPanel
    object EdName: TEdit
```
