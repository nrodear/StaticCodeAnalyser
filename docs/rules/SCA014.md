## Description

Numeric literals embedded directly in business logic carry no meaning: `if RetryCount > 3` forces every reader to guess what 3 represents and forces every maintenance change to find every occurrence by hand. Extract a named constant so the value has a documented purpose and exactly one source of truth. The detector skips trivials (0, 1, 2, -1, common power-of-2 sizes, indexing into known-tiny collections) - see [Detectors] MagicTrivials for the active list. Loop bounds against `Length(arr)` and array index 0 are also ignored. Use the global config to add project-specific trivials (e.g. a fixed PageSize).

## Examples

### Bad (triggers the rule)

```pascal
if RetryCount > 3 then ...
```

### Good (idiomatic fix)

```pascal
const MAX_RETRIES = 3;
if RetryCount > MAX_RETRIES then ...
```
