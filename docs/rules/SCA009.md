# SCA009 — Object created without protective try/finally

![Severity](https://img.shields.io/badge/severity-Warning-orange) ![Type](https://img.shields.io/badge/type-Code%20Smell-yellow)

> .Create call without surrounding try/finally - leak risk on exception

## Metadata

| **Tags** | `memory`, `exception-safety` |

## Description

Similar to MemoryLeak but checked structurally: any .Create followed by code without an enclosing try/finally is flagged regardless of whether Free is called eventually.

## Examples

### Bad (triggers the rule)

```pascal
obj := TFoo.Create;
obj.DoStuff;
obj.Free;
```

### Good (idiomatic fix)

```pascal
obj := TFoo.Create;
try obj.DoStuff finally obj.Free end;
```
