# SCA089 — Public field in class

![Severity](https://img.shields.io/badge/severity-Hint-blue) ![Type](https://img.shields.io/badge/type-Code%20Smell-yellow)

> Public field breaks encapsulation - use a property

## Metadata

## Description

A public field exposes the underlying storage; callers can mutate it without the class noticing. A property gives the class the option to add validation/normalization later without breaking callers. Matches SonarDelphi communitydelphi:PublicField.

## Examples

### Bad (triggers the rule)

```pascal
type TFoo = class
  public
    Count: Integer;
  end;
```

### Good (idiomatic fix)

```pascal
type TFoo = class
  private
    FCount: Integer;
  public
    property Count: Integer read FCount write FCount;
  end;
```
