# SCA093 — Multiple inherited calls in one method

![Severity](https://img.shields.io/badge/severity-Warning-orange) ![Type](https://img.shields.io/badge/type-Bug-red)

> Two or more `inherited;` in the same method - parent side-effects run twice

## Metadata

| Field | Value |
|---|---|
| **Rule ID** | `SCA093` |

| **Tags** | `inheritance`, `bug-risk` |

## Description

Each `inherited` invocation runs the parent implementation. Two calls fire side-effects (event notifications, refcount updates, log lines) twice. Almost always a copy-paste bug. Matches SonarDelphi communitydelphi:TwiceInheritedCalls.

## Examples

### Bad (triggers the rule)

```pascal
procedure Foo;
begin
  inherited;
  DoStuff;
  inherited;
end;
```

### Good (idiomatic fix)

```pascal
procedure Foo;
begin
  inherited;
  DoStuff;
end;
```
