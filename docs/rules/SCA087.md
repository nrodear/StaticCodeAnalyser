# SCA087 — Empty visibility section in class

![Severity](https://img.shields.io/badge/severity-Hint-blue) ![Type](https://img.shields.io/badge/type-Code%20Smell-yellow)

> `public`/`private`/... section header with no members

## Metadata

| Field | Value |
|---|---|
| **Rule ID** | `SCA087` |
| **Kind** | `EmptyVisibilitySection` |
| **Default severity** | Hint |

| **Tags** | `dead-code`, `refactor` |

## Description

A class with a visibility keyword followed immediately by another visibility keyword (or `end`) is usually a refactor leftover - all members of that section moved elsewhere. Delete the empty header. Matches SonarDelphi communitydelphi:EmptyVisibilitySection.

## Examples

### Bad (triggers the rule)

```pascal
type TFoo = class
  public
  private
    FX: Integer;
  end;
```

### Good (idiomatic fix)

```pascal
type TFoo = class
  private
    FX: Integer;
  end;
```
