# SCA028 — DFM event handler references missing method

![Severity](https://img.shields.io/badge/severity-Error-red) ![Type](https://img.shields.io/badge/type-Bug-red)

> OnClick / On... points to a method that does not exist in the form class

## Metadata

| Field | Value |
|---|---|
| **Rule ID** | `SCA028` |
| **Kind** | `DfmDeadEvent` |
| **Default severity** | Error |
| **Type** | Bug |
| **Detector unit** | `uDfmDeadEvent.pas` |
| **Tags** | `dfm`, `streaming`, `dead-code` |

## Description

DFM streaming crashes at form-load time with 'class TForm has no published method X'. Usually caused by a manual rename in the .pas without updating the .dfm.

## Examples

### Bad (triggers the rule)

```pascal
object btnSave: TButton
  OnClick = btnSaveClick   // method removed in .pas
end
```

### Good (idiomatic fix)

```pascal
(restore the method or clear OnClick in the designer)
```
