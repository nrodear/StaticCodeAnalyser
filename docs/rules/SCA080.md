# SCA080 — Redundant Exit/Continue/Break before end

![Severity](https://img.shields.io/badge/severity-Hint-blue) ![Type](https://img.shields.io/badge/type-Code%20Smell-yellow)

> `Exit;` / `Continue;` / `Break;` directly before `end` is a no-op

## Metadata

| Field | Value |

| **Tags** | `dead-code`, `control-flow` |

## Description

If `Exit` is the last statement of a procedure (or `Continue`/`Break` the last in a loop body), control flow already leaves the block - the jump statement is dead. Delete it to clarify intent. Matches SonarDelphi communitydelphi:RedundantJump.

## Examples

### Bad (triggers the rule)

```pascal
procedure Foo;
begin
  DoStuff;
  Exit;
end;
```

### Good (idiomatic fix)

```pascal
procedure Foo;
begin
  DoStuff;
end;
```
