# SCA037 — Duplicate TabOrder among siblings

![Severity](https://img.shields.io/badge/severity-Hint-blue) ![Type](https://img.shields.io/badge/type-Code%20Smell-yellow)

> Two sibling controls in the same parent share the same TabOrder value

## Metadata

| Field | Value |
|---|---|
| **Rule ID** | `SCA037` |
| **Kind** | `DfmTabOrderConflict` |
| **Default severity** | Hint |
| **Type** | Code Smell |
| **Detector unit** | `uDfmTabOrderConflict.pas` |
| **Tags** | `dfm`, `ux` |

## Description

VCL serialisation tolerates duplicate TabOrder but tab navigation becomes order-of-declaration dependent and unpredictable for the user. Renumber so TabOrder is unique per parent.

## Examples

### Bad (triggers the rule)

```pascal
EdA.TabOrder = 0
EdB.TabOrder = 0   // same parent
```

### Good (idiomatic fix)

```pascal
EdA.TabOrder = 0
EdB.TabOrder = 1
```
