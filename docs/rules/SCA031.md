# SCA031 — DFM component without published field

![Severity](https://img.shields.io/badge/severity-Error-red) ![Type](https://img.shields.io/badge/type-Bug-red)

> Component in DFM has no matching published field in the form class

## Metadata

| Field | Value |
|---|---|
| **Rule ID** | `SCA031` |
| **Kind** | `DfmSchemaMismatch` |
| **Default severity** | Error |
| **Type** | Bug |
| **Detector unit** | `uDfmSchemaMismatch.pas` |
| **Tags** | `dfm`, `streaming` |

## Description

DFM streaming requires every named component to have a corresponding published field in the host class. A missing field crashes form construction with EReadError.

## Examples

### Bad (triggers the rule)

```pascal
// .dfm contains object btnSave: TButton, but TForm1 has no btnSave: TButton field
```

### Good (idiomatic fix)

```pascal
(add the field to the published section of TForm1)
```
