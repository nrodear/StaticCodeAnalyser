# SCA003 — SQL string built via concatenation

## Description

Building SQL via 'WHERE x=' + user_input enables SQL injection if the input is untrusted. Use parameterized queries (Params.ParamByName(...).Value := ...) instead.

## Examples

### Bad (triggers the rule)

```pascal
Query.SQL.Text := 'SELECT * FROM Users WHERE Name=''' + UserName + '''';
```

### Good (idiomatic fix)

```pascal
Query.SQL.Text := 'SELECT * FROM Users WHERE Name=:n';
Query.ParamByName('n').AsString := UserName;
```
