# SCA027 — Duplicate (DataSource, DataField) binding

![Severity](https://img.shields.io/badge/severity-Warning-orange) ![Type](https://img.shields.io/badge/type-Bug-red)

> Two or more controls bind the same (DataSource, DataField) pair

## Metadata

| Field | Value |
|---|---|
| **Rule ID** | `SCA027` |
| **Kind** | `DfmDuplicateBinding` |
| **Default severity** | Warning |
| **Type** | Bug |
| **Detector unit** | `uDfmDuplicateBinding.pas` |
| **Tags** | `dfm`, `db-binding` |

## Description

When the user edits one bound control, the second receives a parallel update from the dataset - racey, hard-to-debug overwrites. Bind each (DataSource, DataField) to exactly one control.

## Examples

### Bad (triggers the rule)

```pascal
DBEdit1.DataField = 'NAME'
DBEdit2.DataField = 'NAME'   // same DataSource
```

### Good (idiomatic fix)

```pascal
(bind 'NAME' to exactly one control)
```
