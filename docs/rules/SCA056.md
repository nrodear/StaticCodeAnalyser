# SCA056 — Master-Detail without MasterFields

![Severity](https://img.shields.io/badge/severity-Error-red) ![Type](https://img.shields.io/badge/type-Bug-red)

> TDataSet has MasterSource set but no MasterFields / IndexFieldNames

## Metadata

| Field | Value |
|---|---|

| **Kind** | `DfmMasterDetailUnlinked` |

| **Tags** | `dfm`, `data-access`, `performance` |

## Description

VCL silently performs a Cartesian product instead of the intended Master-Detail join - every parent row pulls every detail row at runtime. Fix by setting MasterFields (and IndexFieldNames for IB/FB).

## Examples

### Bad (triggers the rule)

```pascal
QDetail.MasterSource = dsMaster   // MasterFields = '' (empty)
```

### Good (idiomatic fix)

```pascal
QDetail.MasterSource = dsMaster   QDetail.MasterFields = 'OrderID'
```
