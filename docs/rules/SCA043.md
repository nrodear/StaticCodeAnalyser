# SCA043 — Component has Action + OnClick

![Severity](https://img.shields.io/badge/severity-Warning-orange) ![Type](https://img.shields.io/badge/type-Bug-red)

> Action and OnClick both set - Action wins, OnClick is dead code

## Description

When a TAction is assigned, VCL routes events through the action object and the OnClick never fires. Pick one or call the OnClick body from the action's OnExecute.

## Examples

### Bad (triggers the rule)

```pascal
object btnSave: TButton
  Action = actSave
  OnClick = btnSaveClick   // never fires
end
```

### Good (idiomatic fix)

```pascal
(remove OnClick or remove Action)
```
