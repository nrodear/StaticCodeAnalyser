# SCA039 — DB component on UI form

![Severity](https://img.shields.io/badge/severity-Hint-blue) ![Type](https://img.shields.io/badge/type-Code%20Smell-yellow)

> TFDQuery / TFDConnection directly on a TForm/TFrame instead of a DataModule

## Metadata

| Field | Value |
|---|---|

| **Tags** | `dfm`, `architecture`, `data-access` |

## Description

Database infrastructure on a UI form couples persistence to presentation - hard to reuse, hard to test. Move to a TDataModule and reference it from the form.

## Examples

### Bad (triggers the rule)

```pascal
object frmInvoice: TForm
  object FDQuery1: TFDQuery ...
```

### Good (idiomatic fix)

```pascal
object dmInvoice: TDataModule
  object FDQuery1: TFDQuery ...
```
